Panduan Input Proker (Operational Update)

Halaman ini menjadi standar input progres proker untuk Desa Banjarsari dan Giripurno. Silakan ikuti format agar rekap mingguan otomatis terbaca dan laporan tersusun rapi.

Dear All, dalam rangka memastikan monitoring progres KKN terdokumentasi secara sistematis, mohon seluruh mahasiswa melakukan pembaruan progres program kerja melalui website KKN BG Portal dengan menginput pada file proker.json di folder desa masing-masing (banjarsari/data/proker.json atau giripurno/data/proker.json). Pembaruan data dilakukan secara seragam pada 4 field utama: status (planned/ongoing/done), progress (0â€“100 tanpa simbol %, khusus status done ditetapkan 100), updatedAt (format YYYY-MM-DD), dan output (minimal 1 tautan bukti kegiatan berupa Drive/Docs/Foto). Selain empat field tersebut, elemen lain pada JSON tidak boleh diubah. Diharapkan seluruh proker sudah memiliki progress terkini dan minimal 1 output link pada minggu berjalan agar halaman rekap otomatis dapat digunakan sebagai laporan monitoring. Terima kasih atas kerja samanya.

Contoh Update yang Benar

Contoh: Proker sudah berjalan dan ada dokumentasi kegiatan. 

{
  "status": "ongoing",
  "progress": 40,
  "updatedAt": "2026-01-21",
  "output": [
    { "label": "Dokumentasi Foto", "link": "https://drive.google.com/..." },
    { "label": "Daftar Hadir", "link": "https://docs.google.com/..." }
  ]
}
Catatan: Jika status sudah done, maka wajib progress = 100.

Kontrol Kualitas (wajib dicek sebelum commit)

  1. Pastikan proker.json tidak error (koma & tanda kutip lengkap).
  2. Minimal setiap proker punya updatedAt yang baru minggu ini.
  3. Minimal 1 bukti link per proker (output boleh bertahap).
  4. Setelah commit, uji rekap.html desa masing-masing.
